# Phase 0 — Playable Demo / Доказательство концепции

> **Тип:** Ускоренный прототип (не входит в оригинальный план 1–37).  
> **Цель:** Получить осязаемый артефакт за 3–4 недели, который можно запустить, поклацать мышкой и клавиатурой, и увидеть реакцию системы.  
> **Мотивация:** Инвесторы, нейросети и пользователи оценивают проект по живому результату, а не по спецификациям.

---

## Цель

Создать минимальное демо, демонстрирующее три ключевых свойства Workspace:

1. **Окно как canvas** — GPU-рендеринг поверх winit.
2. **Живой ввод** — мышь и клавиатура немедленно отражаются на экране.
3. **Command Bar — нулевой уровень UI** — плашка ввода, подтверждающая концепцию единой строки управления.

---

## Язык и стек

- **Язык:** Rust
- **Ключевые зависимости:**
  - `winit` 0.30 — окно и события
  - `wgpu` 22 — GPU-рендеринг
  - `cosmic-text` — текстовый рендеринг (шейдерный, GPU-accelerated)
  - `pollster` — блокирующий async для wgpu
- **Целевые ОС:** Windows 10/11 (primary), Linux Wayland/X11 (secondary)

---

## Зависимости

- Phase 1 (Host Shim Windows) — переиспользуем `host_shim` crate, но без полной абстракции `HostBackend`. Демо напрямую работает с winit/wgpu.
- Phase 9 (Display Server Core) — частично переиспользуем `display_server` crate, но в упрощённом виде.

**Важно:** Phase 0 — это **одноразовый прототип**. Код демо **не эволюционирует** в продукт. После завершения Phase 0 код демо архивируется в `archive/demo/`, а уроки интегрируются в Phase 1–11.

---

## Часть системы

**Level -1 — Playable Demo** — временный слой, расположенный между Host Shim и Display Server.

```
User Input (winit)
        ↓
+----------------------------+
|  Demo Controller           |
|  - cursor position         |
|  - click → spawn circle    |
|  - keyboard → print glyph  |
|  - Command Bar state       |
+----------------------------+
        ↓
+----------------------------+
|  wgpu Render Loop          |
|  - clear background        |
|  - draw circles            |
|  - draw cursor quad        |
|  - draw text glyphs        |
|  - draw Command Bar panel  |
+----------------------------+
        ↓
  Swapchain → Display
```

## Требования

### 0.1 Окно и рендер-цикл
- Создание окна 1280×720 через winit.
- Инициализация wgpu Instance → Adapter → Device → Queue.
- Surface с Swapchain (Double Buffering, VSync).
- Рендер-цикл с бюджетом 16.67 мс (60 FPS).
- Обработка Resize: пересоздание Surface configuration.
- Обработка Close: graceful shutdown (drop device, exit process).

### 0.2 Фон и курсор
- Очистка фона в фирменный цвет Workspace (тёмно-синий `#0a0e1a`).
- Курсор — белый квадрат 16×16 px, центрированный на позиции мыши.
- Курсор обновляется каждый кадр по событию `WindowEvent::CursorMoved`.

### 0.3 Взаимодействие мышью
- Левый клик (`MouseInput::Pressed(Left)`) — рисуется круг радиусом 20 px в точке клика.
- Круги остаются на экране (накопительный эффект).
- Правый клик — очищает все круги.
- Скролл — изменяет размер курсора (мин 8 px, макс 64 px).

### 0.4 Взаимодействие клавиатурой
- Печатаемые символы (`ReceivedCharacter`) — отображаются на экране в виде «облака текста» рядом с курсором.
- Backspace — удаляет последний символ.
- Enter — переносит строку.
- Escape — очищает текстовое облако.
- Поддержка кириллицы (русская раскладка) и латиницы.

### 0.5 Command Bar (минимальный)
- Нижняя панель высотой 48 px, цвет `#1a1e2e`.
- Строка ввода с мигающим курсором `|`.
- При вводе текста — live отображение в строке.
- При нажатии `Enter` — введённый текст логируется в консоль и в `%LOCALAPPDATA%\Workspace\demo.log`.
- Горячая клавиша `Shift+Space` — фокус на Command Bar (в демо просто мигает рамка).

### 0.6 Логирование и отладка
- `tracing` для внутренних событий.
- FPS-counter в заголовке окна (`Workspace Demo — 60 FPS`).
- Лог-файл с историей ввода.

## Ключевые функции

| Функция | Описание | Тест |
|---------|----------|------|
| `demo::run()` | Главный цикл демо | Запускается, не паникует 5 сек |

| `command_bar_input()` | Строка ввода снизу | Текст набирается и отображается |

## Интеграция

### Входы
- `winit::event::WindowEvent` — события окна и ввода.
- `winit::event_loop::EventLoop` — управление жизненным циклом.

### Выходы
- `wgpu::SurfaceTexture` — кадр, отправляемый на дисплей.
- Лог-файл `%LOCALAPPDATA%\Workspace\demo.log`.

### Зависимости от других этапов
- **Phase 1 (Host Shim)** — демо переиспользует `events.rs` (InputEvent, KeyState, MouseButton) и `window.rs` (WindowConfig) как типы, но не использует `HostBackend` trait.
- **Phase 9 (Display Server)** — демо использует `wgpu` напрямую, а не через `w-display-server`. Это сознательное упрощение для скорости.

## Критерии приёмки

1. **Запуск:** `cargo run --bin demo` открывает окно без паники.
2. **FPS:** Стабильные 60 FPS на Windows с GPU.
3. **Курсор:** Квадрат-курсор плавно следует за мышью, без задержек.
4. **Клики:** Левый клик рисует круг, правый — очищает.
5. **Клавиатура:** Латиница и кириллица отображаются на экране.
6. **Command Bar:** Нижняя панель видна, текст вводится, Enter логирует.
7. **Graceful exit:** Закрытие окна (Alt+F4 или крестик) завершает процесс без утечек.
8. **Panic Gesture:** `Ctrl+Shift+Esc` или тройное нажатие `Esc` мгновенно закрывает демо (заглушка для будущего PanicExit).
