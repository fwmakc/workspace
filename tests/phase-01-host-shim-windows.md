# Тесты: Этап 01 — Host Shim: Windows

> Оконная подсистема, ввод, жизненный цикл, платформенные абстракции Windows. Все тесты на реальном железе или VM с GPU passthrough.

---

### TC-01-001: Создание окна 800×600
**Тип:** E2E | **Платформа:** Windows | **Данные:** Реальные | **Приоритет:** P0
**Требование:** Phase 01, Key Functions, строка 1
**Предусловия:** Windows 10/11 x64, один монитор 1920×1080, `cargo build --release`.
**Шаги:**
1. Запустить `target/release/core-host-shim.exe`.
2. Дождаться инициализации `winit`.
3. Получить handle окна через `FindWindowW`.
4. Вызвать `GetWindowRect`.
**Ожидаемый результат:**
- Handle ≠ null. Ширина 800 ± 2 px, высота 600 ± 2 px (допуск на DPI scaling).
- `IsWindowVisible(handle)` = true. Заголовок содержит "CORE OS".
**Автоматизация:** автоматический (WinAPI через `windows-sys`).

### TC-01-002: Resize окна 800→1200
**Тип:** E2E | **Платформа:** Windows | **Данные:** Реальные | **Приоритет:** P0
**Требование:** Phase 01, Key Functions, строка 2
**Шаги:**
1. Открыть окно 800×600.
2. Вызвать `SetWindowPos(handle, 0, 0, 0, 1200, 800, SWP_NOMOVE | SWP_NOZORDER)`.
3. Подождать 100 мс. Прочитать `GetWindowRect`.
**Ожидаемый результат:**
- Ширина 1200 ± 2 px, высота 800 ± 2 px.
- В логе `host-shim-windows.log`: `Resize(1200, 800)`.
**Автоматизация:** автоматический.
