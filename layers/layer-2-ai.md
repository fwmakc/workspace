# Layer 2 — ИИ-слой и Intent API (Core.Mind)

Как искусственный интеллект работает в CORE OS: от голосового управления и генеративного UI до локального инференса и облачного моста.


## Принцип: ИИ как невидимый системный оператор

ИИ в CORE — не чат-бот, не помощник, не "Сири". Это автоматика, которая связывает приложения и систему через типизированные команды.

---

## Архитектура Core.Mind

### Semantic Kernel
ИИ понимает всё, что происходит в системе. Индексирует не только файлы, но и действия.

### Intent API
Разработчику не нужно писать код для каждой голосовой команды. Он описывает Намерения (Intents) своего приложения:

```typescript
os.mind.registerIntent({
  name: "create_note",
  description: "Создает новую текстовую заметку",
  parameters: { content: "string" },
  action: (data) => { /* логика создания */ }
});
```

Результат: пользователь говорит "Запиши, что я забыл купить молоко" → система сама вызывает `create_note` из приложения заметок, даже если оно закрыто.

---

## ИИ как "Мост данных"

### Связывание приложений без участия юзера
- "Запомни этот момент" (в игре) → скриншот + распознавание объектов → заметка-черновик в рабочем пространстве
- "Найди все договоры подряда за прошлый месяц, где неустойка выше 5%" → ИИ лезет в файлы, индексирует нативно → список за секунду

### Multi-modal
ИИ "видит" экран не как пиксели, а как структурное дерево компонентов:
- "Перенеси число из этого чека в мою таблицу расходов"
- Прямой доступ к объектам на холсте → выцепление данных → переброска

---

## Фоновое выполнение (Contextual Actions)

ИИ дергает методы приложений напрямую, не разворачивая окна:

| Команда | Действие |
|---------|----------|
| "Сделай музыку тише" | Сигнал в Core.Audio, плеер не открывается |
| "Запиши последние 30 секунд экрана" | Display Server выгружает буфер → файл с тегом |
| "Найди где в этой игре лежит артефакт X" | ИИ парсит гайд в замороженном браузере → зачитывает в наушники |

## Голосовой слой (Zero UI)

### Локальный Whisper
- Работает на отдельном ядре процессора или NPU
- Слушает даже в Exclusive Mode (игры)
- Распознает не слова, а намерения (intents)

### Режим "Второй пилот"
В игре (Exclusive Mode) руки и глаза заняты — единственный канал голос:
- "Скинь этот кадр Ване" → скриншот → мессенджер
- "Поставь будильник на 7" → системный вызов
- "Что там по проекту?" → ИИ читает сводку в наушники

---

## Локальность vs Облако

### Small Language Models (SLM)
- Llama 3 / Phi — запускаются локально через WebGPU
- Простые команды и поиск по файлам — на твоем железе
- Данные не улетают в OpenAI без ведома

### Cloud Bridge
Для сложных задач ("напиши эссе на 5 страниц"):
- Система спрашивает: "Отправить запрос в облачный Бэк-офис?"
- Пользователь решает

---

## Smart Scheduler (Управление ресурсами)

ИИ следит, чтобы фоновые задачи не просаживали FPS:
- Видит нагрузку на GPU
- Динамически ограничивает аппетиты фоновых задач
- "Отрендери это видео, пока я играю" → "Сделаю, но на 30% мощности, чтобы игра не лагала"

---

## Generative UI (On-the-fly)

Если готового приложения нет:
1. ИИ собирает данные из разных источников
2. Пишет JS-код визуализации на лету
3. Отрисовывает временный виджет в пространстве

Пример: "Покажи график моих трат на кофе за год" → ИИ генерирует виджет из данных чеков.

---

## ИИ НЕ находится в ядре

Важное разделение:
- **Ядро (Kernel, VFS):** Только чистая математика (CRDT). Детерминировано. Если не может свести данные → создает именованную ветку (как git branch)
- **ИИ (User Space):** Видит ветки, может предложить пользователю разрешение конфликта. Но ядро никогда не доверяет ИИ критические решения

---

## ИИ-диспетчер (The Guided Experience)

ИИ не болтает, он действует:
- "Я оптимизировал твое пространство"
- "Я нашел ту фотку из 2015-го"
- Автоматический переход в Exclusive при запуске тяжелого 3D-редактора
- Гашение уведомлений при обнаружении фокусировки
