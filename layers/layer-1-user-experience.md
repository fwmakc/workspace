# Layer 1 — User experience | Пользовательский опыт

Это самый верхний уровень описания системы. Как её видит и использует человек. Никаких технических терминов — только то, что происходит на экране и в жизни пользователя.

---

## Что такое CORE OS

CORE OS — это среда, которая заменяет привычный рабочий стол. Человек устанавливает её как обычную программу на любое своё устройство (Windows, Mac, Linux, Android) и получает готовое пространство для работы, жизни и творчества.

Его данные, приложения, настройки — всё следует за ним с устройства на устройство. Не потому что "синхронизируется через облако", а потому что это одна система на всех его экранах.

### CORE и хост-ОС

CORE не требует удаления вашей текущей системы. Она сосуществует с ней:

- **На десктопе** (Windows, macOS, Linux): CORE запускается как приложение на весь экран и заменяет рабочий стол. Хост-ОС остаётся в фоне — драйверы, обновления, совместимость оборудования работают как прежде.
- **На телефоне** (iOS, Android): CORE — это приложение. Оно не может заменить системный лаунчер из-за ограничений Apple/Google, но внутри своего окна предоставляет полноценный Space.
- **На выделенном железе** (Core Base, Raspberry Pi): CORE — единственная система. Нет хост-ОС, нет «под ней».

---

## Первый запуск

1. Скачивает CORE
2. Устанавливает
3. Открывает
4. Вводит своё имя
5. Готово

Никаких "выберите тему", "настройте язык", "установите драйверы", "создайте аккаунт Microsoft". Чистый экран с поисковой строкой.

При первом запуске создаётся единственное пространство "Home" на весь экран. Позже пользователь добавляет новые.

## Space — пространства жизни

### Что такое Space

Жизнь человека разделена на контексты. Работа, семья, хобби, фриланс — каждый контекст со своими файлами, контактами, приложениями.

Space — это осмысленное пространство на экране, заменяющее абстрактные "рабочие столы" обычных ОС. Каждый Space — конкретный контекст: "Работа", "Дом", "Семья", "Проект X". Окна внутри Space принадлежат этому контексту. Нет свалки окон — есть структура.

Один экран может содержать несколько Space одновременно. Данные между ними изолированы.

### Размещение Space

Space заменяет концепцию "плиток" (tile management). Каждый Space занимает определённую область экрана.

**Один Space на все экраны:**

```
┌───────────────────────────────┬───────────────────────────────┐
│                               │                               │
│                               │                               │
│         Space "Работа"        │                               │
│                               │                               │
│                               │                               │
└───────────────────────────────┴───────────────────────────────┘
          Монитор 1                          Монитор 2
```

Для: видеомонтаж, 3D-редакторы, полноэкранная работа.

**Каждый Space на своём экране:**

```
┌───────────────────┬───────────────────┐
│                   │                   │
│  Space "Работа"   │   Space "Дом"     │
│                   │                   │
│  [🔍] Поиск...    │   [🔍] Поиск...   │
│                   │                   │
└───────────────────┴───────────────────┘
      Монитор 1           Монитор 2
```

**Space на части экрана (сплит):**

```
┌───────────────────┬───────────────────┐
│                   │                   │
│  Space "Работа"   │   Space "Дом"     │
│                   │                   │
│  [🔍] Поиск...    │   [🔍] Поиск...   │
│                   │                   │
├───────────────────┤                   │
│                   │   Плеер —         │
│  Space "Семья"    │   Spotify         │
│                   │                   │
│  [🔍] Поиск...    │                   │
│                   │                   │
└───────────────────┴───────────────────┘
```

Разделитель между Space перетаскивается. Пользователь сам решает, сколько места занимает каждый Space.

**Один Space на несколько экранов:**

Space раскидан по нескольким мониторам. Например, таймлайн видеоредактора на одном мониторе, превью — на другом.

### Виды Space

- **Личный** — свой собственный. "Home" по умолчанию, или подключённый к своему Core Base. Полный контроль.
- **Корпоративный** — подключение к Бэку компании. Определённые права, определённые ограничения.
- **Общий** — семейный, командный. У каждого свои права, общие проекты видны всем, личные — только владельцу.
- **Временный** — на время проекта. Фрилансер подключился к Бэку заказчика. Закончив — отключил.
- **Фоновый** — без графической области. Не занимает экран, но работает: качает файлы, принимает сообщения. Пользователь видит только уведомления.

### Состояния Space

- **Открыт** — занимает область экрана, окна рендерятся, строка работает
- **Скрыт** — окна убраны, но Space подключён. Мгновенно возвращается
- **Фоновый** — не имеет области на экране, работает за кадром
- **Оффлайн** — связь с источником данных потеряна, работает с кэшем
- **Отключён** — отключён вручную, кэш сохранён, можно переподключиться
- **Принудительно отключён** — доступ отозван владельцем

### Визуальное отличие Space

Пользователь всегда видит, в каком Space находится:

- **Фоновая подложка** — каждый Space имеет свой цвет или паттерн фона. Настраивается пользователем: сплошной цвет, градиент, паттерн, изображение
- **Метка в Command Bar** — слева в строке отображается имя или иконка текущего Space:

```
[🏠] [🔍] Поиск...          [🎤]
 ^имя Space
```

- При нескольких Space на экране — визуальная граница (разделитель) и разный фон сразу дают понять, где какой контекст

### Управление Space

- **Создать** — новый Space, пустой
- **Подключить** — подключиться к чужому Space (Бэку). Три способа:
  - **Код приглашения** — владелец генерирует код (например, `CORE-A7X9-K2M4`). Ввести в Command Bar: `connect CORE-A7X9-K2M4`. Подходит для корпоративного онбординга (отправил по email)
  - **QR-код** — владелец показывает QR на экране, пользователь сканирует камерой. Подходит для быстрого подключения (настройка семейного Space)
  - **Прямой адрес** — ввести адрес Бэка: `connect 192.168.1.100` или `connect myserver.core.app`. Подходит для продвинутых пользователей (подключение к своему серверу)
- **Скрыть / Показать** — убрать с экрана или вернуть
- **Отключить** — отключить вручную
- **Перетащить** — изменить размер области, перенести на другой монитор
- **Закрыть** — отключить с очисткой кэша

## Строка — элемент управления Space

У каждого Space — своя Command Bar. Строка живёт внутри графической области Space. Нет панели задач. Нет меню "Пуск". Нет лотка уведомлений. Есть строка — и это всё.

### 8 режимов строки

Слева внутри строки — иконка текущего режима. Переключается автоматически по мере ввода, или вручную кликом / горячей клавишей.

| Иконка | Режим | Что делает | Пример |
|--------|-------|-----------|--------|
| 🔍 | Поиск | Поиск по проектам, файлам, тегам, истории | "фото со стройки" |
| 📝 | Заметка | Быстрая запись мысли, идеи | "купить молоко" |
| ⏰ | Напоминание | Планировщик: напомни, запланируй, задача | "напомни завтра позвонить" |
| 🧮 | Калькулятор | Визуальный: формулы, графики, история | "sin(45) + 2^10" |
| 💻 | Терминал | Командная строка, выполнение команд | "$ cargo build" |
| 🌐 | Браузер | Адресная строка + поиск в интернете | "youtube.com" |
| 💬 | Мессенджер | Сообщения, почта, чаты, звонки | "@ivan привет" |
| 🤖 | ИИ-агент | Сложные задачи, анализ, автоматизация | "собери инфу по проекту" |

### Как система определяет режим

**По формату ввода** — системе не нужно объяснять, что ты хочешь:

| Что ввёл | Система поняла | Режим |
|----------|---------------|-------|
| `@ivan` | Это человек → мессенджер | 💬 |
| `user@mail.domain` | Это email → почта | 💬 |
| `+79991234567` | Номер телефона → мессенджер (звонок как опция) | 💬 |
| `youtube.com` | Это сайт → браузер | 🌐 |
| `*.pdf` или `файл*` | Поиск файлов по маске | 🔍 |
| `$ команда` или `> команда` | Терминальная команда | 💻 |
| `2+2` | Это формула → калькулятор | 🧮 |
| `напомни завтра...` | Это команда → планировщик | ⏰ |
| Всё остальное | Поиск + ИИ-агент | 🔍 / 🤖 |

**Голосом явно** — первое слово задаёт контекст:
- "Запиши: купить молоко" → 📝 Заметка
- "Найди фото..." → 🔍 Поиск
- "Проверь почту" → 💬 Мессенджер

**Вручную** — клик по иконке или горячая клавиша (Tab / 1-8)

### Если система не угадала

Показывает топ-3 варианта под строкой. Стрелками выбираешь нужный. Система запоминает — в следующий раз сразу делает правильно.

### Подсказки при вводе

Набираешь текст — под строкой выпадает список подходящих приложений и действий. Первое (самое подходящее) уже выделено — Enter запустит его. Стрелками выбираешь другое.

```
"музыка"
├── 🎵 Core Player (выделено) — Enter для запуска
├── 🌐 youtube.com — открыть в веб-окне
└── 🔍 "музыка" — поиск по проектам
```

### Калькулятор — не атавизм

Не "введите число". Живой инструмент:
- Ввёл формулу → отображается красиво (как в математической нотации)
- Результат сразу виден
- Можно построить график функции
- История вычислений внутри проекта
- Можно сослаться на результат из заметки или задачи

### Мессенджер, почта, звонки — режим строки

- `@ivan` → всегда показывается выпадающий список (даже если одно совпадение): контакты, платформы (Telegram, Slack, почта), последние сообщения. Выбор из списка → открывается полное окно чата в проекте
- `user@mail.domain` → открывает написание письма
- `+79991234567` → мессенджер по умолчанию, звонок как опция
- "проверь почту" → строка показывает непрочитанные прямо в выпадающем списке
- "что писал Алексей на прошлой неделе" → поиск по переписке
- "скинь документ Ивану" → ИИ-агент находит документ и отправляет
- Полноценный мессенджер можно открыть как приложение внутри проекта

### Терминал — через `$` или `>`

- `$ cargo build` → выполняет команду, результат в выпадающем списке или в окне терминала внутри проекта
- `> ls -la` → то же самое
- Полноценный терминал можно открыть как приложение внутри проекта

### Веб-страницы = приложения. Нет отдельного "браузера"

Набрал `youtube.com` в строке → сайт открылся как окно прямо на рабочем столе проекта. Не нужно сначала "открыть браузер", потом вводить адрес.

Граница между "приложением" и "сайтом" стёрта. YouTube — это приложение. Google Docs — это приложение. Они живут в проекте, переключаются как окна, закрываются как окна.

- Каждое веб-окно со своими вкладками. Ссылки открываются как вкладки внутри того же окна — не создают новых окон
- Новое веб-окно — только через Command Bar (набрал адрес → открылось отдельное окно)
- Режим инкогнито — через контекстное меню окна (или "incognito youtube.com")
- Панель разработчика — F12 как обычно
- Можно закрепить часто используемый сайт как иконку на главном экране

### Настройки строки

По умолчанию — **внизу** области Space. Но положение настраивается:

- Вертикально: вверху / внизу
- Горизонтально: по центру / слева / справа
- Ширина: адаптивно / на всю ширину
- Высота: минимальное и максимальное число строк
- Размер шрифта
- Отступы внутри (padding) и снаружи (margin)
- Скругление углов
- Цвет фона, картинка фона, прозрачность

## Проект = Рабочий стол (внутри Space)

Проект — это не папка с файлами. Это **зона** — полноценный рабочий стол со своим смыслом, своими окнами, своим layout, своим контекстом. Проекты живут внутри Space.

### Что внутри проекта

- Открытые приложения с их состоянием (редактор, плеер, терминал)
- Layout окон (где какое окно, какого размера)
- Файлы, задачи, идеи, теги
- Связи между всеми сущностями

### Сессии

- Проект сохраняется автоматически: окна, позиции, состояния приложений
- Можно создать "чистую" сессию — закрыл, и она стёрлась без следа
- Выключил комп, включил завтра или на телефоне — те же рабочие столы, те же окна на тех же местах

### Пример

```
Проект "ЖК Скандинавия" (внутри Space "Работа")
├── [Открыто] Редактор — договор подряда (курсор на строке 42)
├── [Открыто] Почта — правки от заказчика
├── [Свернуто] Плеер — музыка для работы
├── Задача: "Согласовать фундамент"
├── Задача: "Отправить смету"
├── Идея: "Использовать другой бетон"
├── Фото со стройки (12 шт, с телефона)
└── Теги: #строительство #договор #срочно
```

### Переключение между проектами

- Нажал кнопку → выбрал проект → вошёл
- Закреплённый проект на главном экране Space → нажал иконку → вошёл
- Горячая клавиша / жест → переключение между последними проектами

### Всегда активный проект

Не бывает состояния "никакой проект не открыт". Ты всегда внутри какого-то проекта. Даже на главном экране Space — это тоже проект (нулевой, "Home"). Строка всегда работает в контексте текущего проекта.

### Версионность — Time Machine для каждого файла

Каждый файл в проекте помнит свою историю. Не нужно нажимать «сохранить как»:

- Автосохранение фиксирует каждое изменение
- Таймлайн файла: откат к любой точке за последние 30 дней (или больше — настраивает Owner)
- Сравнение версий: посмотреть, что изменилось между двумя состояниями
- Удалил случайно — восстановил за секунду

В Command Bar: «история файла Смета.pdf» → таймлайн → выбрал версию → открыл.

## Теги вместо иерархии

Один файл — много тегов — много точек входа:

- Файл "смета.pdf" с тегами #строительство и #бюджет
- В проекте "ЖК Скандинавия" он виден через тег #строительство
- В проекте "Бюджет семьи" — через тег #бюджет
- Это не копии. Это один и тот же файл, видимый из разных контекстов

### Связанность

Всё связано со всем:

- Задача ссылается на файл
- Файл ссылается на письмо
- Письмо ссылается на контакт
- Идея ссылается на задачу
- Можно перейти от задачи к файлу к письму к человеку — по цепочке связей

### Smart Folders

Папка = сохранённый поисковый запрос. Не нужно раскладывать файлы вручную — Smart Folder сама собирает всё, что подходит под условие.

- "Все PDF за 2025 год" = тег #pdf + дата 2025
- "Всё по проекту X" = тег #проект-x
- "Непрочитанные" = статус "не прочитано"

Smart Folder живёт внутри проекта, обновляется автоматически. Пользователь создаёт её через Command Bar: "новая умная папка: #фото и #отпуск".

## Профили — переключение между жизнями

Профили живут внутри Space. Один Space — несколько профилей.

- **Работа:** Только рабочие проекты, корпоративные инструменты, история только по работе
- **Личное:** Фото, сериалы, мессенджеры, хобби
- **Фриланс:** Отдельные проекты "шабашек", не перемешанные с основной работой
- **Анонимный:** Временный профиль. Закрыл — все данные стёрлись

Переключение между профилями — мгновенное, как вкладки в браузере. Неактивный профиль заморожен: не жрёт CPU, но фоновые сообщения подтягиваются и ждут, когда ты вернёшься.

Для переключения между контекстами жизни целиком (работа / дом / семья) — используй Space. Профили — для переключения внутри одного Space.

---

## Голос — второй способ управления

Руки заняты, глаза заняты — говоришь:

- "Скинь этот документ Ивану" — не открывая мессенджер
- "Запиши экран и сохрани в проект" — не нажимая кнопок
- "Сделай музыку тише" — не выходя из игры
- "Запомни: купить краску для ванной" — заметка создана, лежит в проекте "Ремонт"

Голос работает всегда — даже в полноэкранной игре, даже когда смотришь фильм. Система слушает и действует.

## Приложения — без установки

Нет `.exe`, нет инсталляторов, нет "далее-далее-готово".

Приложение — это адрес. Набрал "калькулятор" — открылся. Закрыл — система сама решает, когда освободить память. Не расходует ресурсы, когда не нужно. Нужен YouTube — открылся нативно, без браузера, без рекламы.

Системные приложения (заметки, калькулятор, плеер, терминал, файлы) — встроены. Остальные — из магазина или по адресу.

Приложения живут внутри проектов. Открыл калькулятор в проекте "Ремонт" — он принадлежит этому проекту, его окно в layout этого рабочего стола.

Приложения не гадят в систему. Удалил — не осталось ни байта мусора.

Приложением может быть любой веб-сайт (открыл как окно) или полноценное нативное приложение. 5 уровней интеграции — от «набрал URL и работает» до «полный натив на WebGPU». Подробно: [Layer Apps](layer-6-apps.md).

### Разрешения приложений

Приложение не получает доступ к данным без спроса. Пользователь всегда видит, что разрешено, и может отозвать в любой момент.

**Где управлять:**
- Command Bar → "права приложений" → список всех приложений с иконками разрешений
- Или: нажал правой кнопкой на окно приложения → "Права доступа"

**Что видит пользователь:**
- Список разрешений: файлы, сеть, контакты, уведомления, микрофон, камера
- Для каждого — переключатель "Разрешено / Запрещено"
- При нажатии на "файлы" — показывается, к каким папкам есть доступ

**Запрос нового разрешения:**
Приложение просит доступ → всплывает окно: "Приложение 'Калькулятор' запрашивает доступ к файлам. Разрешить?" → две кнопки: "Разрешить" и "Запретить". Если пользователь нажал "Запретить" — приложение получает ошибку и должно работать без этой функции.

## Полноэкранный режим приложения

Любое приложение может запросить весь экран. Это **не отдельный режим интеграции с хост-ОС**, а особый layout окна внутри Managed Space.

**Что происходит:**
- Окно приложения растягивается на весь монитор
- Рамки и декорации убираются
- Command Bar сворачивается в тонкую индикатор-линию (или полностью скрывается)
- Весь ввод отдан приложению

**Выход из полноэкрана:**
- `Esc` — стандартный выход
- Системный жест (например, свайп от верхнего края или угловой жест)
- Голосовая команда: «выйди из полноэкрана»

**Panic Gesture:**
- Тройное касание в углу или жест «домой» — мгновенный выход из полноэкрана, возврат в Managed Space
- Работает даже если приложение зависло — Input Router перехватывает на уровне Host Shim

**Не ломает архитектуру:**
- Display Server продолжает рендерить
- Host Shim не отдаёт эксклюзивный доступ к GPU
- Если Shell упал — окно остаётся на экране (shadow state), восстанавливается при перезапуске Shell

## Темы и оболочки

Пользователь может менять внешний вид системы целиком — не только обои, а поведение и стиль всех элементов.

### Системный магазин стилей

Разработчик выпускает дизайн-пак — другие приложения подхватывают автоматически. Не нужно переписывать приложение: `<Button>` из `@core/ui` сам перерисуется под новую тему.

- «Офисный» — строгие линии, приглушённые цвета.
- «Киберпанк» — неон, тени, анимации.
- «Терминал Fallout» — пиксели, зелёный фосфор.

### Автоматическая адаптация

Один и тот же компонент выглядит по-разному:
- На телефоне — больше, под палец.
- На десктопе — компактнее, под мышь.
- В «офисном» режиме — строгий.
- В «киберпанке» — светится неоном.

### Альтернативные оболочки

Пользователь может заменить стандартный Shell на сторонний — как KDE vs GNOME в Linux. Система поддерживает, но не контролирует. Лучшие оболочки вырастают из сообщества: «Битва оболочек» = бесплатный R&D для системы.

Cross-reference: технические детали уровней UI — в [Layer Apps, Уровень 5](layer-6-apps.md). Рендеринг и Display Server — в [Layer Подсистемы](layer-8-technical-decomposition.md).

## Устройства — одна система

Домашний комп, рабочий ноут, телефон, планшет — это не разные устройства. Это **одна система на разных экранах**.

### Как это работает

- Фильм на домашнем компе доступен на телефоне. Не "загружен в облако" — стримится напрямую с домашнего компа
- Заметка с телефона появляется на ноутбуке
- Документ с работы — дома
- Начал смотреть сериал дома, продолжил в метро с той же секунды

### Домашний комп = личное облако

Не платишь $9.99/мес за iCloud. Место ограничено только размером жёсткого диска. Можно купить отдельную "коробочку" (Core Base) и воткнуть в розетку — это всегда включённый узел твоей сети.

### Флешки, диски, внешние носители

Воткнул флешку → уведомление «Обнаружен USB-накопитель» → «Импортировать файлы» / «Сделать бэкап» / «Игнорировать». Файлы копируются в проекты, получают теги, находятся через поиск. CD/DVD — то же самое, только read-only.

Подробно: типы носителей, импорт, экспорт, автоматизация — в [Layer Devices](layer-5-devices.md).

### Ленивая загрузка — файлы без границ

На телефоне видна вся медиатека, но файлы весят 0 байт («призраки»). Нажал Play — домашний Core Base стримит поток напрямую. Умный кэш: система видит, что ты слушаешь один плейлист, и сама качает его для офлайна.

Файл недоступен офлайн — серый значок с облачком. Доступен локально — обычный значок. Всё прозрачно, не требует действий.

### Зеркальные папки — работа рядом с Windows

Проект в CORE можно привязать к обычной папке на диске: `D:\Work`, `~/Documents`. Работаешь в CORE — файл появляется в папке Windows. Коллега сохраняет документ в папку — он тут же виден в проекте CORE с тегами и поиском.

CORE не заменяет привычную папку — надстраивает её. Коллега без CORE видит обычные файлы. Ты внутри CORE видишь ту же папку, но с версионностью, тегами и голосовым поиском.

## Уведомления

Все уведомления из всех Space собираются в единый поток. Каждое уведомление помечено иконкой и цветом своего Space — пользователь сразу видит, из какого контекста пришло.

### Приоритеты Space

Пользователь настраивает приоритет уведомлений для каждого Space:

- **Прерывающие** — звук, вибрация, всплывающее уведомление. Для важных контекстов (работа, семья)
- **Тихие** — только значок в индикаторе, без звука. Для некритичных контекстов (хобби, новости)
- **Молчащие** — уведомления не показываются. Для фоновых Space (скачивание, синхронизация)

Настройка по умолчанию: первый (основной) Space — прерывающий, остальные — тихие.

### Центр уведомлений

Вся история уведомлений доступна через Command Bar ("уведомления") или жест (свайп с верхнего края).

- Список уведомлений сгруппирован по Space и времени (сегодня, вчера, ранее)
- Нажатие на уведомление переводит в соответствующий Space и открывает источник (например, чат с Иваном)
- Можно очистить историю одного Space или всю сразу
- Непрочитанные уведомления помечаются точкой, исчезают после прочтения или перехода

## Безопасность — без паролей

Один раз вошёл в CORE — и всё. Все приложения внутри уже авторизованы.

Система хранит ключи сама. Приложение просит "дай доступ к GitHub" — система делает запрос за тебя, отдаёт только результат. Приложение **никогда не видит** пароль.

Данные зашифрованы. Находятся у пользователя — на его дисках, в его квартире. Не на серверах корпораций.

Технические детали безопасности — шифрование, аутентификация, RBAC, аудит, изоляция, векторы атак — описаны в [Layer Security](layer-7-security.md).

---

## Чего пользователь НЕ делает

- Не устанавливает программы
- Не ищет файлы по папкам
- Не вводит пароли
- Не платит за облако для своих данных — всё хранится на своих устройствах. Сторонние облачные сервисы (VDS, S3) — подключаются опционально, если пользователю это необходимо
- Не настраивает синхронизацию
- Не боится потерять данные при поломке устройства
- Не привязан к конкретному железу
- Не управляет окнами на пустом столе — он управляет проектами внутри Space
- Не перемешивает контексты жизни — каждый контекст в своём Space

## Иерархия сущностей

```
CORE OS
├── Space "Дом"          ← личное пространство
│   ├── Профиль "Я"
│   │   ├── Проект "Ремонт"
│   │   ├── Проект "Хобби"
│   │   └── Home Project
│   └── Профиль "Гость"
│
├── Space "Работа"       ← корпоративное пространство
│   └── Профиль "Я"
│       ├── Проект "ЖК Скандинавия"
│       ├── Проект "Бюджет"
│       └── Home Project
│
└── Space "Семья"        ← общее пространство
    └── Профиль "Я"
        ├── Проект "Отпуск 2025"
        └── Home Project
```

**Space** → содержит **Профили** → каждый профиль содержит **Проекты** → каждый проект содержит окна, файлы, задачи, теги.
